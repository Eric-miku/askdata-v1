## 已跑通业务流程

1. 数据库发现或管理员注册受控 SQLite 数据源。
2. 自然语言问题经过结构化理解、Schema 检索、术语口径匹配、NL2SQL、安全校验、执行和自动修复。
3. 会话检查点保存上一轮结构化指标、维度、筛选、时间范围、排序和 TopN，支持继承、覆盖与显式清除。
4. 查询结果展示表格和推荐图表，支持柱状图、横向条形图、折线图和饼图手动切换。
5. 生成范围、Top/Bottom、结构占比、环比、同比、IQR 异常和轻量趋势预测，每条结论保留数据证据。
6. CSV/XLSX 导出重新执行受控只读 SQL，处理 Excel 公式注入；PNG 从当前 ECharts 实例导出。
7. 管理员在线维护业务术语、别名、指标公式、字段映射、示例问法、发布状态和历史版本。
8. 管理员在线注册、测试、同步、启停和删除 SQLite 数据源配置。
9. 管理员按用户配置数据源、表、字段、静态行过滤、查询和导出白名单；元数据、查询、回放、执行计划和导出统一执行策略。
10. 会话持久化包含用户归属，列表、详情、更新、重置、删除和查询续接均隔离；切换数据库不复用旧库 SQL。
11. SQL 执行限制 JOIN 数、子查询深度、系统对象、查询/导出行数、结果字节和墙钟时间，并输出慢查询警告与稳定错误码。
12. 术语知识支持最多 1000 条 JSON 批量 upsert、逐条错误报告以及 JSON/CSV 导出；批量导入统一进入草稿审核流程。
13. SQLite Schema 同步持久化表、列、主键、外键、索引、DDL 和 SHA-256 指纹，重复同步报告新增、删除和变更表；管理页面可展开 Catalog。
14. SQL 面板可请求受当前用户对象权限保护的 `EXPLAIN QUERY PLAN`，展示执行步骤和确定性索引候选；系统不自动修改数据库索引。

## 安全和运维证据

- SQL AST 统一阻止写操作、DDL 和多语句。
- 表级行过滤使用受限 AST 语法，并对查询、回放、JOIN/CTE、导出和执行计划统一注入；普通响应不暴露内部策略 SQL。
- 管理写接口在配置 `ADMIN_API_TOKEN` 后要求 `X-Admin-Token`。
- 数据源路径被限制在配置的 BIRD databases 目录中。
- 导出日志只记录 SQL SHA-256、数据库、格式和行数，不记录查询结果。
- 每个 HTTP 响应包含 `X-Request-ID`，日志记录状态码和耗时。
- 查询审计关联 request ID、trace ID、用户、会话、数据源、SQL 哈希、结果行数、重试数、耗时和错误码，不记录 SQL 参数或结果明文。
- 未处理异常返回稳定的 `INTERNAL_ERROR` 和 request ID，内部异常细节仅进入服务日志。
- `/health`、`/ready`、`/metrics` 可用于容器和监控探针。
- Docker 后端和前端均使用非 root 用户，Compose 使用只读根文件系统和持久化状态卷。

## 尚需部署方完成的正式验收项

- 当前执行环境没有 Docker CLI，因此镜像和 Compose 需要在安装 Docker 的机器执行最终构建。
- MySQL/PostgreSQL 驱动、凭据管理器和公司只读账号需要结合真实公司环境接入；当前在线管理范围为受控 SQLite。
- 本版本已提供本地用户及数据源/表/字段/静态行过滤/导出策略；SSO、角色组同步、可信身份头注入和身份属性驱动的动态策略仍需连接公司的身份系统。
- OLS 预测用于跑通产品流程，不替代经过业务验证的生产时间序列模型。
- relaxed 准确率最终值必须使用公司的固定真实业务题集和已确认指标口径重新评测。

这些限制不影响当前原型的端到端演示，但不能替代公司环境的安全与业务签字。
